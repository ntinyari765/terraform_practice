Terraform CLI reference

The top 15 Terraform commands you need to know

setup
terraform init
Downloads providers, configures backend, initializes modules.
Use -upgrade to force provider version upgrades past the lock file.
setup
terraform validate
Checks syntax and internal consistency without connecting to any remote service.
Runs entirely locally. Use in CI before plan.
setup
terraform fmt
Formats code to canonical HCL style.
Use -recursive for subdirectories. Use -check in CI pipelines.
plan
terraform plan
Shows proposed changes without touching real infrastructure.
Always save with -out=file.tfplan. Never skip this step.
deploy
terraform apply
Creates or updates infrastructure to match configuration.
Always apply from a saved plan file. Use -auto-approve to skip prompt.
deploy
terraform destroy
Removes all infrastructure managed by the current configuration.
Equivalent to terraform apply -destroy.
read
terraform output
Reads output values from the current state.
Use -json to return outputs in JSON format for scripting.
state
terraform state list
Lists all resources currently tracked in state.
Useful for quickly seeing what Terraform manages in a workspace.
state
terraform state show
Shows all recorded attributes of a specific resource in state.
Use when you need to inspect current values without opening the state file.
state
terraform state mv
Moves a resource to a new address in state without touching real infrastructure.
Use when refactoring module structure to avoid destroy and recreate.
state
terraform state rm
Removes a resource from state. Real infrastructure is completely untouched.
Use when you want Terraform to stop managing a resource.
state
terraform import
Imports existing infrastructure into state. Does NOT generate .tf configuration.
Write the resource block first. One resource per command.
workspace
terraform workspace
Manages workspaces — list, new, select, show, delete.
workspace new creates AND switches to the new workspace.
read
terraform providers
Shows providers required by the current configuration and their versions.
Use terraform providers lock to update the lock file.
read
terraform graph
Outputs a dependency graph of all resources in DOT format.
Pipe to Graphviz to visualize complex dependency chains.
